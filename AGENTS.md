# AGENTS.md

## 1. 目的与适用范围

本文件用于约束代理在 `zt-react-milkdown` 仓库内的协作方式，确保改动聚焦、可解释、低歧义。

适用范围：

- 仓库根目录及所有子目录。

规则优先级：

1. 用户显式需求（最高优先级）
2. 本文件约束
3. 通用工程建议

## 2. 核心强制项（comment-first-tsjs）

当任务涉及 TS/JS 代码文件（`.ts`、`.tsx`、`.js`、`.jsx`、`.mjs`、`.cjs`）改动时，必须使用 `comment-first-tsjs` skill。

执行时至少遵循以下规则：

- 仅补齐缺失注释，不重写已有有效注释。
- 新增注释默认使用中文。
- 函数定义与枚举定义使用 `/** ... */` 风格注释。
- 变量定义与枚举成员使用 `// ...` 风格注释。

## 3. Do / Don’t

### Do

- 先阅读与任务直接相关的文档、源码与配置，再实施改动。
- 坚持最小改动原则，只修改与当前需求直接相关的内容。
- 在交付中清楚说明改动了哪些文件，以及行为变化是否存在风险。

### Don’t

- 不做与当前需求无关的重构、依赖调整或目录结构改造。
- 不擅自扩展需求范围、公共接口或业务行为。
- 不以“规范优化”为由改写已有有效注释或大面积改动无关文件。
- 不执行任何开发命令。

## 4. 交付要求（免强制检验命令）

每次任务完成后，交付说明至少包含：

- Changed：已修改文件路径与变更要点。
- Risk：潜在兼容性或行为变化风险；若无明显风险，明确写“无新增高风险项”。

说明：

- 本仓库不在本文件中设置“必须执行”的固定检验命令。
- 可根据任务需要选择性执行校验，并在交付中说明是否执行及原因。
